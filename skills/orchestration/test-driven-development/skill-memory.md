# Skill Memory: Test-Driven Development

## Purpose
This skill enforces Red-Green-Refactor with an absolute Iron Law: any code written before a failing test must be deleted and restarted, without exception. It provides an extensive rationalization table and red flags list specifically to prevent agents from rationalizing their way out of TDD discipline.

## Category notes
Classified as orchestration because TDD governs how implementation work is structured and sequenced throughout any development workflow.

## Relationships
- systematic-debugging: TDD is used in Phase 4 of debugging to create the failing test before fixing
- subagent-driven-development: subagents are instructed to follow TDD for each task
- verification-before-completion: TDD's red-green verification step aligns with this skill
- writing-plans: plans include explicit TDD steps as part of task structure
