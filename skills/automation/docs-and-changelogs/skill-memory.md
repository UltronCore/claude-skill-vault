# Skill Memory: Docs and Changelogs

## Purpose
Manages project documentation lifecycle: parses git commit history using Conventional Commits format to generate structured CHANGELOG.md files, scaffolds ADR files for architectural decisions, and creates PRD templates for feature planning. Can be integrated into release workflows for automated changelog updates.

## Category notes
Belongs in misc because it covers documentation and project management concerns that span multiple project types and do not fit squarely into a single technical domain like automation or UI.

## Relationships
Can be integrated with github-actions-ci-workflow to automate changelog generation in release pipelines.
