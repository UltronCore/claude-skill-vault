# Docs and Changelogs

**Category:** misc
**Status:** active
**Version:** v1

## What it does
Generates comprehensive changelogs from Conventional Commits git history, maintains CHANGELOG.md with semantic versioning, and scaffolds structured project documentation including Architectural Decision Records (ADR) and Product Requirement Documents (PRD).

## When to use
- Generating a CHANGELOG.md from a git repository using Conventional Commits
- Creating a new Architectural Decision Record to document a significant technical choice
- Scaffolding a PRD template for a new feature or project
- Automating changelog updates as part of a release workflow

## What it produces
Produces CHANGELOG.md entries formatted to Keep a Changelog standards, ADR markdown files with standard sections, PRD templates, and optionally release notes for GitHub releases.
