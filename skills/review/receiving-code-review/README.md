# Receiving Code Review

**Category:** review
**Status:** active
**Version:** v1
**Source:** superpowers@claude-plugins-official

## What it does
Establishes a technically rigorous process for evaluating and responding to code review feedback. Requires understanding and verification before implementing any suggestion, pushes back with technical reasoning when feedback is incorrect, and forbids performative agreement or blind implementation.

## When to use
- After receiving code review feedback from any source
- Before implementing review suggestions, especially unclear or questionable ones
- When external reviewer feedback may conflict with existing architectural decisions

## What it produces
Verified, technically sound implementation of accepted feedback items, with reasoned pushback documented for rejected items.
