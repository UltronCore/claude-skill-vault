# Skill Memory: Receiving Code Review

## Purpose
This skill defines the anti-performative-agreement protocol for handling code review feedback. It establishes that external feedback is a suggestion to evaluate, not an order to follow, and that technical verification must precede any implementation. It also defines the GitHub thread reply mechanism for inline PR comments.

## Category notes
Classified as review because it governs the response side of the code review process.

## Relationships
- requesting-code-review: the dispatching side of the same workflow
- verification-before-completion: complementary — verify before claiming fixes are complete

## Maintenance notes
The "your human partner" phrasing is used throughout this skill as a generic term for the supervising human in the development workflow — no personal data present.
